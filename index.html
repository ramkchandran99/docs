Git Tag Commands

Tags are ref's that point to specific points in Git history. Tagging is generally used to capture a point in history that is used for a marked version release (i.e., v1.0.0).

Creating Tags
CREATE
git tag tagName
Creates a lightweight tag (pointer to a specific commit) for the current branch you've checked out.
Annotated Tags
CREATE
git tag -a tagName -m "message"
Creates an annotated tag with a message. Annotated tags store extra metadata including the tagger name, email, and date.
Listing Tags
LIST
git tag
Lists all tags in the current repository in alphabetical order.
Show Tag Info
LIST
git show tagName
Shows information about a specific tag, including the tagger, date, and the associated commit.
Push Single Tag
PUSH
git push origin tagName
Pushes a specific tag to the remote repository named "origin".
Push All Tags
PUSH
git push --tags
Pushes all tags from your local repository to the remote repository.
Delete Local Tag
DELETE
git tag -d tagName
git tag --delete tagName
Deletes a tag from your local repository. Both commands work identically.
Delete Remote Tag
DELETE
git push origin -d tagName
git push origin --delete tagName
Deletes a tag from the remote repository. Both commands work identically.
Delete Multiple Tags
DELETE
git tag -d tagName1 tagName2
Deletes multiple tags from your local repository in a single command.
Delete Multiple Remote Tags
DELETE
git push origin -d tagName1 tagName2
Deletes multiple tags from the remote repository in a single command.

Important Notes

1. Lightweight tags are just pointers to specific commits, while annotated tags are stored as full objects in the Git database with their own checksum, tagger info, and message.

2. Tags are not automatically pushed to remotes when you run git push. You must explicitly push tags using git push --tags or git push origin tagName.

3. Deleting a tag locally doesn't delete it from remote repositories, and vice versa. You need to delete tags in both places separately if needed.
